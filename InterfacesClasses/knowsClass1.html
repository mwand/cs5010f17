Every Object Knows Its Class (1)

Here are two objects of the same class C1:



x = 10
y = 20
r = 14



x = 15
y = 35
r = 5

These objects also have a this field, but we don't ordinarily show the this field.

The class definition of C1 says every object of that class has three fields named x, y, and r: 

      class C1 {
          int x;
          int y;
          int r;
      
          public int foo () { return x + y; }
          public int bar (int n) { return r + n; }
          ...
      }

The class definition also defines two methods, named foo and bar, that are applicable to any object of this class.
