Contracts and Purpose Statements for Methods 

      class Bomb implements Widget {
          ...
          // RETURNS: a bomb like this one, but
          //     as it should be after a tick
          // DETAILS: the bomb moves vertically

The DETAILS part of the purpose statement refines the purpose statement given in the Widget interface. 

          public Bomb afterTick () {
              return new Bomb (x, y + BOMB_SPEED);
          }
      }

Bomb implements the Widget interface, so the value returned by afterTick is a Widget. So afterTick satisfies its contract.

This method is so simple it doesn't need examples.
